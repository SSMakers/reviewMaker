# UI/UX Proposal: Review Image Upload

## Design Goal

사용자가 이미지 URL을 직접 만들지 않아도 리뷰 이미지가 포함된 Cafe24 리뷰를 등록할 수 있게 합니다. 현재 앱의 장점인 "엑셀 하나로 일괄 등록" 흐름은 유지하되, 이미지가 있는 사용자에게 추가 선택지를 제공합니다.

## Brand Direction

- 신뢰감 있는 업무 자동화 도구: 과한 장식보다 명확한 상태, 오류 복구, 진행률을 우선합니다.
- 쇼핑몰 운영자를 위한 실무형 UI: 사용자가 반복 작업을 줄였다는 느낌을 즉시 받아야 합니다.
- 톤: "간단하지만 안전한 도구". 버튼과 상태 문구는 기술 용어보다 작업 결과 중심으로 표현합니다.

## Current UX Problem

- 현재 엑셀 `하이퍼링크` 컬럼은 이미 공개 URL이 있는 사용자에게만 편합니다.
- 로컬 이미지만 가진 사용자는 블로그/호스팅에 수동 업로드하고 URL을 복사해야 합니다.
- 사용자는 "어떤 이미지가 어떤 리뷰에 붙는지"를 등록 전에 확인하기 어렵습니다.

## Proposed Main Screen Changes

### 1. File Section

기존:

- `엑셀 파일 선택`

제안:

- `엑셀 파일 선택`
- `이미지 폴더 선택`
- 선택 상태 표시:
  - `엑셀: reviews.xlsx`
  - `이미지 폴더: /.../review-images, 38개 이미지 발견`

### 2. Image Mapping Mode

이미지 매칭 방식을 명확히 선택하게 합니다.

- `엑셀 URL 사용`: 기존 `하이퍼링크` 컬럼의 공개 URL만 사용
- `파일명으로 매칭`: 엑셀에 `이미지파일명` 컬럼을 추가하고 선택한 폴더에서 찾기
- `URL 우선, 없으면 파일명`: 기존 고객과 신규 고객 모두에게 가장 안전한 기본값

권장 기본값: `URL 우선, 없으면 파일명`

### 3. Preflight Check

`리뷰 등록 시작` 전에 짧은 검사를 실행합니다.

검사 결과 예시:

- 총 리뷰 100건
- URL 이미지 42건
- 업로드 필요 이미지 51건
- 이미지 없음 7건
- 필수값 누락 2건

오류가 있으면 등록 버튼을 막고, 경고는 사용자가 진행할 수 있게 합니다.

### 4. Progress States

기존 진행률은 리뷰 전송 중심입니다. 이미지 업로드가 추가되면 단계를 나눕니다.

1. 엑셀 분석
2. 이미지 매칭 확인
3. 이미지 업로드
4. Cafe24 리뷰 등록

로그에는 실패한 행 번호와 파일명을 함께 보여줍니다.

## Excel Contract Proposal

기존 컬럼은 유지합니다.

- `하이퍼링크`: 공개 이미지 URL. 기존 엑셀 호환을 위해 URL이 아닌 파일명이 들어오면 선택한 이미지 폴더에서 찾습니다.

신규 선택 컬럼:

- `이미지파일명`: 선택한 이미지 폴더 안의 파일명. 예: `review_001.jpg`

매칭 규칙:

- `하이퍼링크`가 공개 URL이면 업로드하지 않고 그대로 사용합니다.
- `하이퍼링크`가 파일명이거나 `이미지파일명`이 있으면 서버에 업로드해 URL을 생성합니다.
- 둘 다 없으면 이미지 없이 리뷰를 등록하거나, preflight 정책에 따라 경고합니다.

## Developer Handoff

- UI 변경 대상: `ui/main_window.py`
- 엑셀 계약 변경 대상: `review_article_builder.py`
- 이미지 업로드 client: `external_api/server/server_api.py`
- 이미지 매칭 정책: `image_mapping.py`
- 사전 검사 요약: `review_preflight.py`
- worker 변경 대상: `api_worker.py`

## Implementation Status

- 메인 화면에 `이미지 폴더 선택`과 `이미지 매칭 방식` 컨트롤을 추가했습니다.
- 기본 매칭 방식은 `URL 우선, 없으면 파일명`입니다.
- 등록 시작 시 사전 검사 결과를 로그에 보여줍니다.
- 서버 `/review/image/upload`로 로컬 이미지를 업로드하고 반환 URL을 Cafe24 리뷰 본문의 `<img>` 태그로 전송합니다.

## Deferred Decisions

- 여러 이미지가 한 리뷰에 들어갈 수 있는지 Cafe24 API 스펙 확인이 필요합니다.
- 서버 URL 생성 API가 준비되기 전까지 앱 구현은 feature flag 또는 disabled UI로 둘 수 있습니다.
- 드래그 앤 드롭 개별 매칭 UI는 강력하지만 구현 비용이 커서 1차 범위에서는 제외하는 것이 좋습니다.
